# Lesson Authoring Guide

Этот документ описывает, как создавать уроки в рамках текущего курса, чтобы материал был последовательным, практичным и легко усваивался.

## Core Principles

1. Один урок = одно концептуальное ядро.
2. Минимально достаточная теория, максимум практики.
3. Каждое изменение кода должно иметь учебную цель.
4. Любая новая абстракция объясняется "зачем".
5. Урок должен быть самодостаточным для повторения.

## Narrative Rules

- Писать на русском, технические термины и код - на английском.
- Использовать обязательную структуру: Context -> Concept -> Step-by-Step -> Code Example -> Verification.
- Избегать абстрактных формулировок без примера из текущего кода.
- Для каждого нового понятия давать минимум один рабочий пример кода (TypeScript, DTO, guard, test, или command snippet).
- Добавлять объяснение "как думать" (decision rationale), а не только "что сделать".
- Добавлять "What To Remember" (3-5 тезисов, не больше).
- Каждому уроку давать практический mini-homework.

## Lesson Depth Standard

- Урок должен быть достаточным, чтобы студент смог повторить шаг без внешних источников.
- Каждый шаг описывается как:
  - цель шага;
  - конкретное изменение;
  - ожидаемый результат.
- Если в уроке есть архитектурный выбор, нужно указать trade-offs.
- Примеры кода должны быть короткими, но исполнимыми по смыслу.
- В уроках roadmap-уровня обязательно добавлять структуру внедрения по шагам.

## Code Change Rules

- Изменения должны быть малыми и атомарными.
- Нельзя смешивать несколько новых концепций в одном шаге.
- Перед введением новой техники проверять, что базовый сценарий работает.
- Каждый урок должен содержать команды проверки (`npm run start:dev`, `npm run test`, HTTP checks).
- Если добавляется инфраструктура (DB, cache, queue), в уроке указывать локальный запуск.

## Lesson Quality Checklist

- Понятно, зачем нужен урок?
- Ровно одно ключевое понятие?
- Есть связь с предыдущими уроками?
- Есть явный список изменений в коде?
- Есть хотя бы один код-пример, объясняющий концепцию?
- Есть пошаговый план внедрения (если это roadmap/проектирование)?
- Есть шаги валидации результата?
- Есть секция "что запомнить"?

## Folder and Naming Convention

- Уроки лежат в `docs/lessons`.
- Формат имени: `lesson-XX-topic.md`.
- Нумерация строгая по возрастанию.
- Новые уроки нельзя создавать без явного подтверждения от студента в текущем диалоге.

## Template Contract

Каждый новый урок обязан основываться на `docs/lessons/lesson-template.md` и включать все обязательные секции без удаления.
