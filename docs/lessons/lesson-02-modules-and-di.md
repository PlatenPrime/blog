# Lesson 02: Modules and Dependency Injection

## Context

После запуска приложения нужно понять базовую архитектуру NestJS: как модули связывают части системы.

## Concept

`Module` группирует компоненты, а DI контейнер управляет созданием и внедрением зависимостей (`providers`).

## Code Changes

- Разобрана структура `AppModule` как корневого контейнера зависимостей.
- Зафиксирован паттерн: `controller` получает `service` через constructor injection.
- Подготовлены правила для будущих модулей (`users`, `auth`, `posts`, `comments`).

## Why This Matters

Понимание DI и модульности позволяет строить расширяемую архитектуру без жесткой связности.

## What To Remember

- Модуль - это организационная единица.
- Provider создается/кешируется DI контейнером.
- Внедрение через конструктор - базовый и предпочтительный способ.

## Verify

- Проследить цепочку `AppController -> AppService`.
- Добавить временный provider и подключить его в модуле.
